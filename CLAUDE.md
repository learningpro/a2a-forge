## Architecture

Tauri 2.x desktop app: Rust backend handles all HTTP/SQLite, React frontend handles UI. They communicate via typed IPC commands generated by tauri-specta.

### Data Flow

```
User clicks Run → TestPanel builds JSON-RPC payload (lib/a2a.ts)
  → calls commands.sendTask() from bindings.ts
  → Rust send_task command (commands/tasks.rs)
  → a2a::client::send_task_rpc() POSTs to {agent_url}/a2a
  → if async (pending/running): auto-polls tasks/get until terminal state
  → returns JSON result to frontend
  → testStore.finishTask() updates per-skill execution state
  → ResponseViewer renders result with smart media detection
```

### Rust Backend (`src-tauri/src/`)

- `lib.rs` — Tauri builder. All commands registered in `collect_commands![]` macro. Auto-exports bindings.ts with `@ts-nocheck` prepended.
- `a2a/types.rs` — AgentCard, AgentSkill, AgentRow, TaskEvent structs. All derive `specta::Type` for TS generation.
- `a2a/client.rs` — HTTP client. `send_task_rpc()` POSTs to `/a2a`, auto-polls async tasks. `build_sse_request()` for streaming.
- `commands/` — 22 Tauri commands across 6 modules: agents (7), tasks (3), history (3), saved_tests (3), workspaces (4), settings (2).
- `db.rs` — SQLite migrations (5 tables: workspaces, agents, history, saved_tests, settings). WAL mode. `get_pool()` helper shared by all commands.
- `credentials.rs` — Keyring-first credential storage with AES-SQLite fallback stub.
- `state.rs` — `AppState` with shared `reqwest::Client`.
- `error.rs` — `AppError` enum (6 variants) with thiserror + specta.

### Frontend (`src/`)

- `bindings.ts` — **Auto-generated by tauri-specta on every `cargo run`**. DO NOT edit. Contains typed command wrappers and A2A types. Returns `Result<T, AppError>` wrappers.
- `lib/tauri-helpers.ts` — `unwrap()` function to extract Result data (throws on error). `HistoryEntry` type. Safe from specta overwrite.
- `lib/a2a.ts` — JSON-RPC payload builders. `skill_id` in `params`, `input` object with skill-specific params.
- `lib/curl.ts` — Generates equivalent curl commands from request data.
- `stores/testStore.ts` — Per-skill concurrent execution state. Key: `"agentId:skillId"` → `SkillExecution` (status, result, chunks, latency).
- `stores/agentStore.ts` — Agent CRUD, selection, per-agent default headers (persisted to SQLite).
- `stores/uiStore.ts` — Theme, panel widths, sidebar collapse state (persisted via Zustand persist).
- `stores/workspaceStore.ts` — Workspace CRUD.
- `components/layout/` — AppShell (3-panel flex), Sidebar, SkillPanel, TestPanel (re-export), ResizeHandle.
- `components/test/` — TestPanel (orchestrator), InputForm (adaptive tabs), ResponseViewer (media detection), JsonTree, HistoryList, SavedTestsList, MonacoWrapper (lazy-loaded), SkillMetadata, TaskStatus.
- `components/agent/` — AddAgentDialog (debounced URL preview), AgentListItem, AgentContextMenu, AgentHeadersDialog.
- `hooks/useTheme.ts` — System dark/light detection + manual override via `data-theme` attribute.
- `hooks/useStreamingTask.ts` — SSE via Tauri Channel.
- `hooks/useKeyboardShortcuts.ts` — Ctrl/Cmd+N (add), Ctrl/Cmd+Enter (run), Ctrl/Cmd+Shift+C (curl).

### Key Patterns

Every Rust command needs both `#[tauri::command]` AND `#[specta::specta]` attributes, plus registration in `collect_commands![]` in lib.rs.

Per-agent default headers are stored in SQLite settings table with key `card:{agentId}:headers` and auto-loaded when selecting an agent.

ResponseViewer deep-scans JSON results for media URLs (`.jpg`, `.mp4`, `.mp3`, `data:image/`, etc.) and renders inline `<img>`, `<video>`, `<audio>` previews.

## Critical Gotchas

**bindings.ts is auto-generated:** tauri-specta regenerates it on every `cargo run` in debug mode. Never put custom code there. Use `src/lib/tauri-helpers.ts` instead. lib.rs auto-prepends `@ts-nocheck`.

**Zustand selector stability — causes infinite render loops:**
- NEVER: `useStore((s) => s.someFunction())` — function call creates new ref each render
- NEVER: `useStore((s) => s.x ?? {})` — `?? {}` creates new object each render
- NEVER: `const { a, b } = useStore()` — subscribes to entire store
- DO: `useStore((s) => s.field)` for individual fields, `useMemo` for derived values

**specta BigInt forbidden:** Never use `i64`/`u64` in command signatures or `#[derive(Type)]` structs. Use `i32`/`u32` for params, `String` for timestamps.

**Dev port is 1421** (not default 1420 — another Tauri app uses it). `beforeDevCommand` in tauri.conf.json auto-kills conflicts.

**A2A request format:** `skill_id` goes in `params` (not `metadata`). `input` object with skill-specific params in `params`. Tasks are async — always poll `tasks/get`. Status flow: `pending` → `running` → `completed`/`failed`.

**SQLite init:** `Database.load("sqlite:workbench.db")` must run in App.tsx before any Tauri commands use the pool.
