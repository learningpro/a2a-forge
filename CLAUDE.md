# A2A Workbench — Development Guide

## Project Overview

Cross-platform desktop app (Tauri 2.x + React + TypeScript) for testing A2A protocol agents.

## Tech Stack

- **Backend:** Rust (Tauri 2.x), reqwest, sqlx, tauri-specta
- **Frontend:** React 18, TypeScript, Vite, Tailwind CSS 4, Zustand 5, @tanstack/react-query
- **Database:** SQLite (via tauri-plugin-sql)
- **Dev port:** 1421 (avoid 1420 — WeChatty uses it)

## Key Files

- `src-tauri/src/lib.rs` — Tauri builder, command registration
- `src-tauri/src/a2a/client.rs` — A2A HTTP client with async polling
- `src-tauri/src/commands/` — All Rust Tauri commands
- `src/bindings.ts` — Auto-generated by tauri-specta (DO NOT edit manually)
- `src/lib/tauri-helpers.ts` — `unwrap()` helper and custom types (safe from specta overwrite)
- `src/stores/` — Zustand stores (agentStore, testStore, uiStore, workspaceStore)


## Known Gotchas

### bindings.ts is auto-generated
- tauri-specta regenerates `src/bindings.ts` on every `cargo run` in debug mode
- NEVER put custom code in bindings.ts — it will be overwritten
- Put helpers in `src/lib/tauri-helpers.ts` instead
- lib.rs auto-prepends `@ts-nocheck` after generation

### Zustand selector stability
- NEVER use `useStore((s) => s.someFunction())` — function calls in selectors create new refs → infinite loop
- NEVER use `useStore((s) => s.something ?? {})` — `?? {}` creates new object every render
- DO use `useMemo` to derive computed values from stable selector results
- DO select individual fields: `useStore((s) => s.field)` not `const { a, b } = useStore()`

### Port conflicts
- Dev port is 1421 (not default 1420 — WeChatty uses that)
- `beforeDevCommand` in tauri.conf.json auto-kills port 1421 before starting
- If port still busy: `lsof -ti :1421 | xargs kill -9`

### A2A request format (AIGC Service)
- `skill_id` goes in `params` (not `metadata`)
- `input` object with skill-specific params (e.g., `{prompt: "..."}`) goes in `params`
- Tasks are async — always poll `tasks/get` after `tasks/send`
- Status values: `pending` → `running` → `completed`/`failed`

### SQLite initialization
- Database must be loaded via `Database.load("sqlite:workbench.db")` in App.tsx before any commands
- Rust commands access pool via `get_pool(&app)` from `db.rs`

### specta BigInt
- Never use `i64`/`u64` in Tauri command signatures or structs with `#[derive(Type)]`
- Use `i32`/`u32` for command params, `String` for timestamps
