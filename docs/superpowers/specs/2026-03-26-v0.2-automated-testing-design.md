# v0.2 Automated Testing — Design Spec

## Overview

为 A2A-Forge 添加自动化测试能力：测试套件、断言验证、CLI 集成、测试报告。让用户从"手动点击测试"进化到"自动化回归测试"。

## 数据模型

### 新增 SQLite 表

```sql
-- 测试套件（两层：agent 级 + workspace 级）
CREATE TABLE test_suites (
    id TEXT PRIMARY KEY,           -- nanoid
    name TEXT NOT NULL,
    description TEXT DEFAULT '',
    agent_id INTEGER,              -- NULL = workspace 级跨 agent 套件
    workspace_id INTEGER NOT NULL,
    run_mode TEXT DEFAULT 'sequential',  -- 'sequential' | 'parallel'
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (agent_id) REFERENCES agents(id) ON DELETE CASCADE,
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE
);

-- 套件中的测试步骤
CREATE TABLE test_steps (
    id TEXT PRIMARY KEY,
    suite_id TEXT NOT NULL,
    sort_order INTEGER NOT NULL,
    name TEXT NOT NULL,
    agent_id INTEGER NOT NULL,     -- 每个步骤绑定具体 agent
    skill_name TEXT NOT NULL,
    request_json TEXT NOT NULL,     -- JSON-RPC payload
    assertions_json TEXT DEFAULT '[]',  -- Assertion[] 序列化
    timeout_ms INTEGER DEFAULT 60000,
    FOREIGN KEY (suite_id) REFERENCES test_suites(id) ON DELETE CASCADE,
    FOREIGN KEY (agent_id) REFERENCES agents(id) ON DELETE CASCADE
);

-- 套件运行记录
CREATE TABLE suite_runs (
    id TEXT PRIMARY KEY,
    suite_id TEXT NOT NULL,
    status TEXT DEFAULT 'running',  -- 'running' | 'passed' | 'failed' | 'cancelled'
    total_steps INTEGER DEFAULT 0,
    passed_steps INTEGER DEFAULT 0,
    failed_steps INTEGER DEFAULT 0,
    duration_ms INTEGER DEFAULT 0,
    started_at TEXT DEFAULT (datetime('now')),
    finished_at TEXT,
    FOREIGN KEY (suite_id) REFERENCES test_suites(id) ON DELETE CASCADE
);

-- 每个步骤的运行结果
CREATE TABLE step_results (
    id TEXT PRIMARY KEY,
    run_id TEXT NOT NULL,
    step_id TEXT NOT NULL,
    status TEXT DEFAULT 'pending',  -- 'pending' | 'running' | 'passed' | 'failed' | 'skipped'
    response_json TEXT,
    assertion_results_json TEXT,    -- AssertionResult[] 序列化
    error_message TEXT,
    duration_ms INTEGER DEFAULT 0,
    FOREIGN KEY (run_id) REFERENCES suite_runs(id) ON DELETE CASCADE,
    FOREIGN KEY (step_id) REFERENCES test_steps(id) ON DELETE CASCADE
);
```

### 断言类型

```typescript
type AssertionType =
  | 'status_equals'        // task status === expected
  | 'json_path_equals'     // $.result.foo === expected
  | 'json_path_exists'     // $.result.foo exists
  | 'json_path_contains'   // $.result.foo contains substring
  | 'json_path_matches'    // $.result.foo matches regex
  | 'response_time_lt'     // duration < threshold_ms
  | 'contains_media'       // response contains image/video/audio URL

interface Assertion {
  id: string;
  type: AssertionType;
  path?: string;          // JSONPath for json_path_* types
  expected?: string;      // expected value
  description?: string;   // human-readable label
}

interface AssertionResult {
  assertion_id: string;
  passed: boolean;
  actual?: string;
  message: string;
}
```

## Rust 后端新增

### 新增 Commands（`commands/suites.rs`）

| Command | 说明 |
|---------|------|
| `create_suite(name, description, agent_id?, workspace_id, run_mode)` | 创建套件 |
| `update_suite(id, name?, description?, run_mode?)` | 更新套件 |
| `delete_suite(id)` | 删除套件（级联删除步骤和运行记录） |
| `list_suites(workspace_id, agent_id?)` | 列出套件 |
| `get_suite(id)` | 获取套件详情（含步骤） |
| `add_step(suite_id, name, agent_id, skill_name, request_json, assertions_json, timeout_ms)` | 添加步骤 |
| `update_step(id, name?, request_json?, assertions_json?, timeout_ms?, sort_order?)` | 更新步骤 |
| `delete_step(id)` | 删除步骤 |
| `reorder_steps(suite_id, step_ids[])` | 重排步骤顺序 |
| `run_suite(suite_id)` | 执行套件，返回 run_id |
| `cancel_suite_run(run_id)` | 取消运行中的套件 |
| `get_suite_run(run_id)` | 获取运行结果 |
| `list_suite_runs(suite_id, limit?)` | 列出历史运行 |
| `export_report(run_id, format)` | 导出 JSON/HTML 报告 |

### 断言引擎（`a2a/assertions.rs`）

```rust
pub fn evaluate_assertions(
    response: &serde_json::Value,
    assertions: &[Assertion],
    duration_ms: i32,
) -> Vec<AssertionResult>
```

- 使用 `jsonpath_lib` crate 做 JSONPath 查询
- 每个断言独立求值，不短路
- 返回全部结果供 UI 展示

### 套件执行器（`a2a/runner.rs`）

```rust
pub async fn run_suite(
    pool: &SqlitePool,
    client: &reqwest::Client,
    suite_id: &str,
) -> Result<String, AppError>  // returns run_id
```

- sequential 模式：逐步执行，失败可选继续或停止
- parallel 模式：tokio::join_all 并发执行所有步骤
- 每步完成后立即写入 step_results
- 支持通过 cancel flag 中断

## CLI 集成

### 方案：Tauri CLI 插件

在 `src-tauri/src/cli.rs` 中添加 CLI 模式：

```bash
# 运行指定套件
a2a-forge run-suite --suite "Image Generation Tests" --workspace default

# 运行所有套件
a2a-forge run-all --workspace default

# 导出报告
a2a-forge run-suite --suite "Smoke Test" --format json --output report.json
a2a-forge run-suite --suite "Smoke Test" --format html --output report.html

# 列出套件
a2a-forge list-suites --workspace default
```

退出码：0 = 全部通过，1 = 有失败，2 = 执行错误

### CI 集成示例

```yaml
# GitHub Actions
- name: Run A2A tests
  run: |
    ./a2a-forge run-all --workspace default --format json --output results.json

- name: Upload test report
  uses: actions/upload-artifact@v4
  with:
    name: a2a-test-report
    path: results.json
```

## 测试报告

### JSON 格式

```json
{
  "suite": "Image Generation Tests",
  "run_id": "abc123",
  "status": "failed",
  "summary": { "total": 3, "passed": 2, "failed": 1 },
  "duration_ms": 12400,
  "started_at": "2026-03-26T10:30:00Z",
  "steps": [
    {
      "name": "Generate landscape",
      "status": "passed",
      "duration_ms": 3200,
      "assertions": [
        { "type": "status_equals", "expected": "completed", "actual": "completed", "passed": true }
      ]
    }
  ]
}
```

### HTML 格式

自包含的单文件 HTML，内嵌 CSS，包含：
- 摘要卡片（通过/失败/总数/耗时）
- 每步展开/折叠的详情
- 断言结果高亮（绿色通过/红色失败）
- 可直接在浏览器打开或作为 CI artifact

- 新增 `stores/suiteStore.ts`
- 新增 `components/suite/` 目录：SuiteList, SuiteEditor, StepEditor, SuiteRunViewer, AssertionEditor
- TestPanel 增加 Test/Suites tab 切换

### Phase 3: CLI + 报告
- 新增 `src-tauri/src/cli.rs`
- tauri.conf.json 添加 CLI 配置

- JSON 报告导出

### Phase 4: 测试 + 验证
- Rust 单元测试（断言引擎、runner）
- 前端单元测试（suiteStore、组件）
- 集成测试（CLI 端到端）
- 使用 /loop 持续验证

## 验收标准

1. 可以创建 agent 级和 workspace 级测试套件
2. 可以添加/编辑/删除/重排测试步骤
3. 每个步骤支持 7 种断言类型
4. 套件可以顺序或并行执行
5. 运行结果实时展示，断言逐条显示通过/失败
6. 可以导出 JSON 和 HTML 报告
7. CLI 模式可以无头运行套件并返回正确退出码
8. 所有新增代码有对应测试
9. `npx tsc --noEmit` 和 `cargo check` 通过
