# v0.3 Local Registry Proxy — Design Spec

## Overview

在 Tauri 进程内嵌一个 HTTP 代理服务器，作为本地 A2A 注册中心。外部客户端可以通过它发现和调用已注册的 Agent，同时支持请求/响应拦截修改、延迟模拟、故障注入、流量录制回放。

## 架构

```
外部客户端 → localhost:9339/a2a → Proxy Server (axum)
                                    ├─ Agent Registry (返回聚合的 agent cards)
                                    ├─ Request Interceptor (修改请求)
                                    ├─ Fault Injector (延迟/错误)
                                    ├─ Traffic Recorder (录制)
                                    └─ Forward to real Agent URL → Response Interceptor → 返回客户端
```

### 核心组件

1. **Proxy Server** — axum HTTP 服务器，监听本地端口（默认 9339）
2. **Agent Registry** — 聚合所有已注册 Agent 的 card，提供 `/.well-known/agent.json` 端点
3. **Interceptor Engine** — 规则引擎，匹配请求并应用修改
4. **Fault Injector** — 延迟注入、错误码注入、超时模拟
5. **Traffic Recorder** — 录制请求/响应对，支持回放

## 数据模型

### 新增 SQLite 表

```sql
-- 拦截规则
CREATE TABLE intercept_rules (
    id          TEXT PRIMARY KEY,
    name        TEXT NOT NULL,
    enabled     INTEGER DEFAULT 1,
    match_type  TEXT NOT NULL,        -- 'all' | 'agent' | 'skill' | 'method'
    match_value TEXT DEFAULT '',       -- agent_id, skill_name, or method name
    action_type TEXT NOT NULL,         -- 'modify_request' | 'modify_response' | 'delay' | 'error' | 'mock'
    action_json TEXT NOT NULL,         -- action-specific config
    priority    INTEGER DEFAULT 0,
    workspace_id TEXT NOT NULL,
    created_at  TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE
);

-- 流量录制
CREATE TABLE traffic_records (
    id            TEXT PRIMARY KEY,
    session_name  TEXT NOT NULL,
    agent_id      TEXT,
    skill_name    TEXT,
    request_json  TEXT NOT NULL,
    response_json TEXT,
    status_code   INTEGER,
    duration_ms   INTEGER,
    timestamp     TEXT DEFAULT (datetime('now')),
    workspace_id  TEXT NOT NULL,
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE
);

-- 代理服务器状态（settings 表中存储）
-- key: proxy_enabled, value: "true"/"false"
-- key: proxy_port, value: "9339"
-- key: recording_session, value: session name or null
```

## 拦截规则 action_json 格式

### modify_request
```json
{
  "headers": { "X-Debug": "true" },
  "body_transform": "$.params.input.prompt = 'modified: ' + $.params.input.prompt"
}
```

### modify_response
```json
{
  "headers": { "X-Proxy": "a2a-forge" },
  "body_transform": "$.result.status.state = 'completed'"
}
```

### delay
```json
{ "delay_ms": 3000, "jitter_ms": 500 }
```

### error
```json
{ "status_code": 500, "body": { "error": "Simulated server error" } }
```

### mock
```json
{
  "response": { "jsonrpc": "2.0", "result": { "status": { "state": "completed" } } }
}
```

## Rust 后端

### 新增模块 `proxy/`

- `proxy/server.rs` — axum HTTP 服务器，启动/停止控制
- `proxy/registry.rs` — Agent 注册中心，聚合 cards
- `proxy/interceptor.rs` — 规则匹配和请求/响应修改
- `proxy/recorder.rs` — 流量录制和回放

### 新增 Commands (`commands/proxy.rs`)

| Command | 说明 |
|---------|------|
| `start_proxy(port?)` | 启动代理服务器 |
| `stop_proxy()` | 停止代理服务器 |
| `get_proxy_status()` | 获取代理状态（运行中/端口/连接数） |
| `create_rule(...)` | 创建拦截规则 |
| `update_rule(...)` | 更新规则 |
| `delete_rule(id)` | 删除规则 |
| `list_rules(workspace_id)` | 列出规则 |
| `toggle_rule(id, enabled)` | 启用/禁用规则 |
| `start_recording(session_name)` | 开始录制 |
| `stop_recording()` | 停止录制 |
| `list_recordings(workspace_id)` | 列出录制会话 |
| `get_recording(session_name)` | 获取录制详情 |
| `replay_recording(session_name)` | 回放录制 |
| `delete_recording(session_name)` | 删除录制 |

### Proxy Server 路由

```
GET  /.well-known/agent.json  → 返回聚合的 agent card（合并所有已注册 agent 的 skills）
POST /a2a                     → 代理转发（应用拦截规则 → 转发 → 应用响应规则 → 返回）
GET  /proxy/status            → 代理状态 JSON
```

### ProxyPanel（新 tab）

TestPanel 的 tab 栏增加第三个 tab: **Proxy**

```
[Test] [Suites] [Proxy]
```

### ProxyPanel 布局

```
┌──────────────────────────────────────────┐
│ Local Proxy          [● Running :9339]   │
│                      [Stop] [Settings]   │
├──────────────────────────────────────────┤
│ [Rules] [Recording] [Traffic]            │

│                                          │

```

#### Rules Tab
- 规则列表，每条显示：名称、匹配条件、动作类型、启用开关
- 新建/编辑规则对话框

#### Recording Tab
- 录制控制：开始/停止按钮，当前会话名
- 录制列表：会话名、条数、时间
- 点击会话查看详情，支持回放

#### Traffic Tab
- 实时流量日志（最近 100 条）
- 每条显示：时间、agent、skill、状态码、耗时
- 点击展开查看完整请求/响应

## 验收标准

1. 代理服务器可启动/停止，监听指定端口
2. `GET /.well-known/agent.json` 返回聚合的 agent card
3. `POST /a2a` 正确转发请求到目标 agent 并返回响应
4. 拦截规则可以修改请求/响应内容
5. 延迟注入和故障注入正常工作
6. 流量可以录制和回放
7. 前端 Proxy tab 可以控制代理和管理规则
8. 所有新增代码有对应测试
