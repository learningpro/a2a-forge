1. Create `src-tauri/src/a2a/mod.rs` with `pub mod types;`

2. Create `src-tauri/src/a2a/types.rs` with these structs (all derive Debug, Clone, Serialize, Deserialize, Type; all use `#[serde(rename_all = "camelCase")]`):

   - `AgentCard`: name (String), description (Option<String>), url (String), version (String), protocol_version (Option<String>), capabilities (AgentCapabilities), skills (Vec<AgentSkill>), default_input_modes (Vec<String>), default_output_modes (Vec<String>), provider (Option<AgentProvider>), documentation_url (Option<String>)
     - Do NOT use `#[serde(flatten)]` with `extra` field — it causes specta compile issues. Instead just skip unknown fields by using `#[serde(default)]` on optional fields.
   - `AgentCapabilities`: streaming (Option<bool>), push_notifications (Option<bool>), state_transition_history (Option<bool>)
   - `AgentSkill`: id (String), name (String), description (Option<String>), tags (Option<Vec<String>>), input_modes (Option<Vec<String>>), output_modes (Option<Vec<String>>), examples (Option<Vec<serde_json::Value>>)
   - `AgentProvider`: organization (Option<String>), url (Option<String>)
   - `AgentRow`: id (String), url (String), nickname (Option<String>), card (AgentCard), last_fetched_at (i64), workspace_id (String)

3. Create `src-tauri/src/commands/agents.rs` with 7 commands. Every command uses dual attributes `#[tauri::command]` and `#[specta::specta]`, is `pub async fn`, and returns `Result<T, AppError>`.

   **fetch_agent_card(base_url: String, state: tauri::State<'_, AppState>) -> Result<AgentCard, AppError>**
   - Construct URL: `format!("{}/.well-known/agent.json", base_url.trim_end_matches('/'))`
   - Use `state.http_client.get(&url).timeout(Duration::from_secs(10)).send().await`
   - On reqwest error: return `AppError::Http("Network error — check the URL: {e}".into())`
   - On non-2xx status: return `AppError::Http("HTTP {status} from agent endpoint".into())`
   - Parse JSON: `resp.json::<AgentCard>().await` — on decode error return `AppError::Serialization("Invalid JSON — not a valid agent card".into())`
   - Return parsed AgentCard

   **add_agent(base_url: String, nickname: Option<String>, workspace_id: String, app: tauri::AppHandle, state: tauri::State<'_, AppState>) -> Result<AgentRow, AppError>**
   - Call fetch_agent_card logic (inline, not recursive command call) to get card
   - Generate UUID with `uuid::Uuid::new_v4().to_string()`
   - Serialize card to JSON string with `serde_json::to_string(&card)?`
   - Get current unix timestamp: `std::time::SystemTime::now().duration_since(UNIX_EPOCH).unwrap().as_secs() as i64`
   - Use `app.state::<tauri_plugin_sql::DbInstances>()` to get the db pool. Call `.0.lock().await` to get the HashMap, then `.get("sqlite:workbench.db")` to get the pool. Use the pool's `execute` method with raw SQL: `INSERT INTO agents (id, url, nickname, card_json, last_fetched_at, workspace_id) VALUES (?, ?, ?, ?, ?, ?)`
   - NOTE: tauri-plugin-sql v2 uses `DbInstances` (not `DbPool`). The type is `tauri_plugin_sql::DbInstances` which wraps `Mutex<HashMap<String, DbPool>>`. Each `DbPool` is an enum. For SQLite, use the `execute` method on the pool variant.
   - If the exact DbInstances API is awkward, fall back to using `tauri::Manager` trait's `app.state()` and work with the raw connection. An alternative safe approach: use `sqlx::SqlitePool` directly by creating your own pool in AppState during setup. But try DbInstances first.
   - Return AgentRow { id, url: base_url, nickname, card, last_fetched_at: now, workspace_id }

   **list_agents(workspace_id: String, app: tauri::AppHandle) -> Result<Vec<AgentRow>, AppError>**
   - Query: `SELECT id, url, nickname, card_json, last_fetched_at, workspace_id FROM agents WHERE workspace_id = ? ORDER BY rowid ASC`
   - Deserialize each row's card_json to AgentCard
   - Return Vec<AgentRow>

   **delete_agent(agent_id: String, app: tauri::AppHandle) -> Result<(), AppError>**
   - Execute: `DELETE FROM agents WHERE id = ?` with agent_id
   - History table has ON DELETE CASCADE, so history rows auto-delete

   **refresh_agent(agent_id: String, app: tauri::AppHandle, state: tauri::State<'_, AppState>) -> Result<AgentRow, AppError>**
   - Query existing agent to get url: `SELECT url FROM agents WHERE id = ?`
   - If not found, return AppError::NotFound
   - Re-fetch card from the url using fetch_agent_card logic
   - Update: `UPDATE agents SET card_json = ?, last_fetched_at = ? WHERE id = ?`
   - Return updated AgentRow

   **export_agents(workspace_id: String, app: tauri::AppHandle) -> Result<String, AppError>**
   - Query all agents in workspace
   - Build JSON array of objects: `[{"baseUrl": url, "nickname": nickname, "workspaceId": workspace_id}, ...]`
   - Return serialized JSON string

   **import_agents(json_data: String, workspace_id: String, app: tauri::AppHandle, state: tauri::State<'_, AppState>) -> Result<Vec<AgentRow>, AppError>**
   - Parse json_data as `Vec<ImportEntry>` where ImportEntry = { base_url: String, nickname: Option<String> }
   - For each entry, call add_agent logic (fetch + insert)
   - Collect results, skip entries that fail (log error), return successful ones

4. Update `src-tauri/src/commands/mod.rs` — add `pub mod agents;` line.

**CRITICAL NOTES on tauri-plugin-sql Rust API:**
The tauri-plugin-sql v2 crate exposes `DbInstances` as managed state. If this type is not directly importable or the API differs, use this alternative approach:
- Add `sqlx = { version = "0.8", features = ["sqlite", "runtime-tokio"] }` to Cargo.toml
- In `state.rs`, add a `db_pool: sqlx::SqlitePool` field to AppState
- In `lib.rs` setup closure, create the pool and set it on AppState
- Commands use `state.db_pool` directly

However, first try the plugin approach since the plugin already manages the pool. Check `tauri_plugin_sql` crate exports to find the right type.
