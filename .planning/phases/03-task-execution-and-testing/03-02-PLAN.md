       - Use `\\\n` line continuation for readability. Strip trailing slash from agentUrl.

    4. **src/stores/testStore.ts**: Create Zustand store (NO persist — ephemeral test state):
       - State: `taskId: string | null`, `status: TaskStatus` (type = "idle" | "running" | "completed" | "failed" | "canceled"), `chunks: TaskChunk[]`, `result: unknown | null`, `latencyMs: number | null`, `startedAt: number | null`, `inputText: string`, `inputTab: "message" | "context" | "headers"`, `responseTab: "rendered" | "raw"`, `customHeaders: Record<string, string>`
       - Actions: `startTask(taskId)` sets running + startedAt, `appendChunk(chunk)`, `finishTask(result, status?)` computes latency, `reset()`, `setInputText(text)`, `setInputTab(tab)`, `setResponseTab(tab)`, `setCustomHeaders(headers)`
       - Export `TaskChunk` type: `{ raw: unknown; status?: { state: string; message?: string }; artifact?: unknown }`

    5. **src/stores/workspaceStore.ts**: Create Zustand store with persist (only activeWorkspaceId persisted):
       - State: `workspaces: Array<{ id: string; name: string; createdAt: number }>`, `activeWorkspaceId: string`
       - Actions: `loadWorkspaces()` calls `commands.listWorkspaces()`, `createWorkspace(name)` calls `commands.createWorkspace(name)` then reloads, `deleteWorkspace(id)` calls `commands.deleteWorkspace(id)` then reloads, `setActiveWorkspace(id)` sets activeWorkspaceId
       - Default activeWorkspaceId = "default"
       - Persist key: "a2a-workspace-state", partialize to `{ activeWorkspaceId }`

    6. **src/hooks/useKeyboardShortcuts.ts**: Create hook per research pattern:
       - `useKeyboardShortcuts(handlers: { onAddAgent?: () => void; onRunTest?: () => void; onCopyCurl?: () => void })`
       - useEffect with keydown listener. Check `e.ctrlKey || e.metaKey`. Cmd+N = onAddAgent, Cmd+Enter = onRunTest, Cmd+Shift+C = onCopyCurl.
       - preventDefault on matched shortcuts.

    7. **src/hooks/useStreamingTask.ts**: Create hook:
       - `useStreamingTask()` returns `{ run(agentUrl, payload, authHeader?) }` async function
       - Creates `new Channel<TaskEvent>()` from `@tauri-apps/api/core`
       - Sets `channel.onmessage = (event) => appendChunk(event)` where appendChunk is from testStore
       - Calls `commands.streamTask(agentUrl, payload, authHeader ?? null, channel)` which returns taskId
       - Calls `startTask(taskId)` at start
       - After streamTask resolves (stream ended), calls `finishTask(lastChunk)` to compute latency

    8. **src/bindings.ts**: Add new command bindings for all 16 new Rust commands. Follow existing pattern:
       - Add `TaskEvent` type: `{ id: string | null; status: { state: string; message: string | null } | null; artifact: unknown | null; raw: unknown }`
       - Add commands: `sendTask`, `streamTask` (with `Channel<TaskEvent>` param), `cancelTask`, `saveHistory`, `listHistory`, `clearHistory`, `saveTest`, `listSavedTests`, `deleteSavedTest`, `listWorkspaces`, `createWorkspace`, `deleteWorkspace`, `setActiveWorkspace`
       - `streamTask` signature: `async streamTask(agentUrl: string, payload: unknown, authHeader: string | null, extraHeaders: Record<string, string> | null, onEvent: Channel<TaskEvent>): Promise<string>`
       - Import `Channel` from `@tauri-apps/api/core` at the top of bindings.ts
       - `saveSetting` already exists (rename from `saveSettings` to `saveSetting` to match Rust fn name if needed, or keep consistent)
